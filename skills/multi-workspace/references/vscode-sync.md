# VS Code Config Sync Reference

## Overview

Multi merges VS Code configuration files from all sub-repos into workspace-root `.vscode/` files. The generated files have a header comment: `"This file is generated by multi. Do not edit directly."`

## Deep Merge Algorithm

For each repo (in order, skipping repos with `skipVSCode: true`):

1. Read the repo's source JSON file (empty `{}` if missing).
2. Apply repo-specific defaults.
3. Deep merge into accumulated result:
   - **Dicts**: recursively merged (later repo values override earlier for scalar keys).
   - **Lists**: concatenated with deduplication preserving order.
4. **`${workspaceFolder}` rewriting**: all `${workspaceFolder}` paths in string values are automatically rewritten to `${workspaceFolder}/repo-name`.

## settings.json

### Merge Process

1. For each repo: merge `settings.json` with `settings.shared.json` from that repo (combined result also written back to the repo's `settings.json`).
2. Apply `vscode.settingsOverrides` from the repo's `multi.json` config.
3. Exclude keys listed in `vscode.skipSettings`.
4. After all repos: merge `.vscode/settings.shared.json` from workspace root (merged last, highest priority for cross-repo settings).
5. Python repos (detected by `pyproject.toml`, `requirements.txt`, etc.) have their names added to `python.autoComplete.extraPaths`.

### settings.shared.json

Place at workspace root `.vscode/settings.shared.json` for settings that should apply across all repos. This file is merged last so its values take highest priority.

## launch.json

### Merge Process

1. Each launch configuration (non-"attach" requests) gets a `cwd` set to `${workspaceFolder}/repo-name` if not already present.
2. Configurations and compounds can be marked "required" via:
   - `"required": true` in the JSON itself, OR
   - `requiredCompounds` / `requiredLaunchConfigurations` in `multi.json` repo config.
3. After all repos: `.vscode/launch.shared.json` from workspace root is merged in (for cross-repo compounds).
4. A **master compound** is auto-created named after the workspace directory (title-cased), containing all required configurations.
5. The master compound sets `preLaunchTask` to `"All Required Tasks - WorkspaceName"`.
6. If a compound with that name already exists, it is renamed to `"... (Original)"`.

### launch.shared.json

Place at workspace root `.vscode/launch.shared.json` for cross-repo launch compounds (e.g., compounds that depend on launch configs from multiple repos).

## tasks.json

### Merge Process

1. Each task gets `options.cwd` defaulted to `${workspaceFolder}` (the sub-repo's folder after rewriting).
2. Tasks can be marked "required" via `"required": true` in JSON or `requiredTasks` in `multi.json`.
3. After all repos: `.vscode/tasks.shared.json` from workspace root is merged in.
4. A **master compound task** is auto-created: `"All Required Tasks - WorkspaceName"` that `dependsOn` all required task labels, running them in `"parallel"`.

### tasks.shared.json

Place at workspace root `.vscode/tasks.shared.json` for cross-repo task definitions.

## extensions.json

All `recommendations` arrays from all repos are concatenated with deduplication.

## .devcontainer

Copied from a sub-repo to the workspace root. Relative paths like `../` are rewritten to `../repo-name/`.

## Key Files

| File | Location | Purpose |
|------|----------|---------|
| `settings.json` | `.vscode/` (root) | Generated merged settings |
| `launch.json` | `.vscode/` (root) | Generated merged launch configs |
| `tasks.json` | `.vscode/` (root) | Generated merged tasks |
| `extensions.json` | `.vscode/` (root) | Generated merged extensions |
| `settings.shared.json` | `.vscode/` (root) | Manual cross-repo settings (merged last) |
| `launch.shared.json` | `.vscode/` (root) | Manual cross-repo launch compounds |
| `tasks.shared.json` | `.vscode/` (root) | Manual cross-repo tasks |

## Troubleshooting

**Launch config not appearing**: Check the source repo's `.vscode/launch.json`. Ensure `skipVSCode` is not `true`. Run `multi sync vscode launch`.

**Settings conflict between repos**: Later repos override earlier ones for scalar values. Use `vscode.settingsOverrides` in `multi.json` for per-repo overrides. Use `vscode.skipSettings` to exclude contentious keys.

**Cross-repo compound not working**: Put it in `.vscode/launch.shared.json` at the workspace root, not in any sub-repo. Reference launch config names as they appear in the merged output (which includes the original repo's names).

**workspaceFolder paths wrong**: Multi automatically rewrites `${workspaceFolder}` to `${workspaceFolder}/repo-name`. Do not manually prefix repo names in source files.
